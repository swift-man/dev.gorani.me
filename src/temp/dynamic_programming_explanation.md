# Dynamic Programming이란?

Dynamic Programming(DP)은 **큰 문제를 작은 부분 문제로 나누고**, 그 **부분 문제의 답을 저장해 재사용**해서 전체 문제를 효율적으로 푸는 방법입니다.

핵심은 두 가지입니다.

1. **겹치는 부분 문제(Overlapping Subproblems)**
   - 같은 계산이 여러 번 반복됩니다.
2. **최적 부분 구조(Optimal Substructure)**
   - 작은 문제의 최적해를 조합해서 큰 문제의 최적해를 만들 수 있습니다.

예를 들어 피보나치 수열을 단순 재귀로 풀면 `fib(5)`를 구할 때 `fib(3)`, `fib(2)` 같은 값이 여러 번 반복 계산됩니다.
DP는 이 값을 한 번 구하면 저장해 두고 다시 씁니다.

```swift
func fib(_ n: Int) -> Int {
    if n <= 1 { return n }
    return fib(n - 1) + fib(n - 2)
}
```

위 코드는 중복 계산이 많습니다.

DP로 바꾸면 이렇게 됩니다.

```swift
func fib(_ n: Int) -> Int {
    if n <= 1 { return n }

    var dp = Array(repeating: 0, count: n + 1)
    dp[1] = 1

    for i in 2...n {
        dp[i] = dp[i - 1] + dp[i - 2]
    }

    return dp[n]
}
```

이제 각 값은 한 번씩만 계산합니다.

---

## 왜 빠를까?

DP를 쓰면 보통:

- 단순 재귀: **지수 시간**
- DP 적용 후: **다항 시간**

으로 줄어드는 경우가 많습니다.

피보나치 예시는:

- 단순 재귀: `O(2^n)` 수준
- DP: `O(n)`

입니다.

---

## 구현 방식 2가지

### 1. Top-Down (Memoization)
재귀로 풀되, 계산한 값을 캐시에 저장합니다.

```swift
func fib(_ n: Int) -> Int {
    var memo = Array(repeating: -1, count: max(2, n + 1))

    func solve(_ x: Int) -> Int {
        if x <= 1 { return x }
        if memo[x] != -1 { return memo[x] }

        memo[x] = solve(x - 1) + solve(x - 2)
        return memo[x]
    }

    return solve(n)
}
```

장점은 원래 점화식을 코드로 옮기기 쉽다는 점입니다.
단점은 재귀 깊이가 깊으면 스택 부담이 생길 수 있습니다.

### 2. Bottom-Up (Tabulation)
작은 문제부터 차례대로 표를 채워 올라갑니다.

장점은 반복문 기반이라 안정적이고 성능 예측이 쉽습니다.
실무에서는 종종 이 방식이 더 선호됩니다.

---

## DP 문제를 보는 방법

DP 문제는 보통 아래 순서로 접근합니다.

### 1. 상태(State)를 정의한다
`dp[i]`, `dp[i][j]`가 무엇을 의미하는지 정합니다.

예:
- `dp[i]` = i번째까지 고려했을 때의 최적값
- `dp[i][j]` = i번째 물건까지 고려하고, 배낭 용량이 j일 때의 최대 가치

### 2. 점화식(Transition)을 만든다
이전 상태에서 현재 상태를 어떻게 만들지 정의합니다.

예:
- `dp[i] = dp[i - 1] + dp[i - 2]`
- `dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - weight] + value)`

### 3. 초기값(Base Case)을 정한다
시작값이 틀리면 전체가 다 틀어집니다.

### 4. 계산 순서를 정한다
점화식이 참조하는 값이 먼저 계산되도록 순서를 잡아야 합니다.

---

## 대표적인 DP 문제

DP는 아래 같은 문제에서 자주 등장합니다.

- 피보나치 수열
- 계단 오르기
- 동전 교환
- 배낭 문제(Knapsack)
- 최장 공통 부분 수열(LCS)
- 편집 거리(Edit Distance)
- 최장 증가 부분 수열(LIS)
- 경로 개수 세기

---

## DP와 분할 정복의 차이

둘 다 문제를 쪼개지만 차이가 있습니다.

- **분할 정복**: 부분 문제가 서로 독립적인 경우가 많음
  - 예: Merge Sort
- **DP**: 부분 문제가 서로 겹침
  - 같은 부분 문제를 여러 번 만나므로 저장해 재사용함

즉, DP는 “쪼개기 + 저장 + 재사용”이 핵심입니다.

---

## DP가 항상 정답은 아니다

모든 문제를 DP로 푸는 건 아닙니다.

- 부분 문제가 겹치지 않으면 굳이 DP가 필요 없을 수 있음
- 상태 정의가 너무 크면 메모리 사용량이 과해질 수 있음
- 그리디가 더 단순하고 빠른 경우도 있음

실제로는 “이 문제가 DP 문제인가?”보다,
**중복 계산이 있는가, 상태를 정의할 수 있는가, 점화식이 나오는가**를 먼저 보는 게 좋습니다.

---

## 한 줄로 정리

Dynamic Programming은
**중복되는 부분 문제의 답을 저장해 두고 재사용함으로써 전체 문제를 효율적으로 푸는 기법**입니다.

# PNG와 JPG의 차이점

핵심은 **압축 방식**과 **용도**입니다.

## 1. 압축 방식
- **PNG**: **무손실 압축**
  - 저장해도 원본 품질이 거의 유지됩니다.
  - 선, 글자, 아이콘, UI 같은 또렷한 이미지에 유리합니다.
- **JPG(JPEG)**: **손실 압축**
  - 용량을 줄이기 위해 일부 정보를 버립니다.
  - 사진처럼 색 변화가 많은 이미지에 유리합니다.

## 2. 화질 특징
- **PNG**
  - 글자, 로고, 도형 경계가 선명합니다.
  - 여러 번 저장해도 열화가 거의 없습니다.
- **JPG**
  - 사진에는 자연스럽지만,
  - 글자나 선이 많은 이미지에서는 깨져 보이거나 번질 수 있습니다.
  - 반복 저장하면 화질이 점점 더 떨어질 수 있습니다.

## 3. 투명 배경 지원
- **PNG**: **지원함**
  - 투명 배경, 반투명(alpha) 표현 가능
- **JPG**: **지원 안 함**
  - 배경이 항상 채워져 있어야 합니다.

## 4. 파일 용량 경향
- **PNG**
  - 단순 그래픽에서는 효율적일 수 있음
  - 사진에서는 보통 JPG보다 커지는 편
- **JPG**
  - 사진 파일을 작게 만들기 좋음
  - 웹 업로드, 썸네일, 일반 사진 공유에 많이 사용

## 5. 언제 무엇을 쓰면 좋은가
### PNG가 좋은 경우
- 앱 아이콘
- 로고
- UI 캡처
- 텍스트가 많은 이미지
- 투명 배경이 필요한 이미지

### JPG가 좋은 경우
- 일반 사진
- 배경 이미지
- 용량을 줄여야 하는 웹용 사진
- SNS 업로드용 사진

## 한 줄 정리
- **PNG = 선명함, 무손실, 투명 배경 가능**
- **JPG = 사진용, 용량 작음, 손실 압축**

실무 기준으로 보면:
- **로고 / 스크린샷 / UI 리소스 → PNG**
- **실사 사진 / 배너 배경 / 용량 중요한 이미지 → JPG**

# AGENTS.md

## Markdown 작성 지침

- 새 `.md` 파일을 만들 때는 먼저 같은 주제 또는 같은 폴더의 기존 `.md` 파일 2~3개를 확인하고, 그 문서들의 스타일을 기준으로 작성합니다.
- 우선적으로 맞춰야 하는 요소:
  - frontmatter 형식 (`title`, `description`, 필요 시 `slug`)
  - 배지 형식과 표기 문자열
  - 제목 계층 (`##`, `###`)과 문단 밀도
  - 리스트 표기 방식 (`*`, 번호 목록)
  - 코드 블록 형식과 메타 (`title` 표기)
    - bash 명령 예시는 반드시 ` ```bash title="shell"` 형식을 사용합니다.
    - TypeScript 예시는 반드시 ` ```ts title="ts"` 형식을 사용합니다.
    - TSX 예시는 반드시 ` ```tsx title="tsx"` 형식을 사용합니다.
    - 다른 언어도 가능하면 언어명과 `title` 메타를 함께 맞춥니다.
  - 링크/강조/설명 문체
- `bash` 코드 블록을 ` ```bash`만 단독으로 쓰지 말고, 항상 ` ```bash title="shell"`로 작성합니다.
- `ts` 코드 블록을 ` ```ts`만 단독으로 쓰지 말고, 항상 ` ```ts title="ts"`로 작성합니다.
- `tsx` 코드 블록을 ` ```tsx`만 단독으로 쓰지 말고, 항상 ` ```tsx title="tsx"`로 작성합니다.
- 같은 폴더의 기존 문서에 배지가 있으면 새 문서도 같은 위치와 같은 패턴으로 작성합니다.
- 배지의 alt 문구, shields.io URL 형식, 로고 파라미터는 기존 문서를 그대로 따르고, 바뀌는 값은 모델 버전처럼 실제로 달라져야 하는 부분만 수정합니다.
- 문서 스타일이 폴더마다 다르면 저장소 전체를 획일화하지 말고, 새 문서는 가장 가까운 기존 문서 스타일을 따릅니다.
- 기존 `.md` 문서를 수정할 때도 해당 문서 주변의 표현 방식과 구조를 유지합니다.
